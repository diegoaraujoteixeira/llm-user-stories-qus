As a web developer, I want the Servers view to display a list of configured servers (e.g., Apache HTTP, Tomcat, and Jetty) with their status (started/stopped) so that I can easily manage and monitor server instances from within the IDE.

As a web developer, I want to start, stop, and restart servers directly from the Servers view with a single click so that I can quickly deploy and test my web applications without leaving the IDE.

As a web developer, I want the ability to add new server runtimes (e.g., specifying installation directory, JRE, and deployment settings) so that I can configure custom server environments for different project requirements.

As a web developer, I want to publish/deploy my project to a server automatically upon save or manually via the Servers view, with support for incremental deployment of changed files, so that I can preview my changes in real-time without full redeployment cycles.

As a web developer, I want the Servers view to display server logs, console output, and deployment status in a dedicated console tab, with filtering and search capabilities, so that I can debug application startup errors and runtime issues effectively.
