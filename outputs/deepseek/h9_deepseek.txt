As a Spring developer, I want to configure Cassandra connectivity (e.g., contact points, port, keyspace, and credentials) using a dedicated XML namespace (<cassandra:cluster> and <cassandra:session>) so that I can set up data access declaratively in legacy Spring XML applications.

As a Spring developer, I want to enable Cassandra repository support using the @EnableCassandraRepositories annotation in JavaConfig so that I can create DAO interfaces that automatically implement common CRUD and query methods.

As a developer, I want to provide Cassandra mapping and conversion settings (e.g., custom converters, naming strategies, and schema actions like create-if-not-exists) via CassandraMappingContext and CassandraConverter beans so that I can fine-tune object mapping without manual XML configuration.

As a developer, I want to configure Cassandra-specific transaction and consistency level settings (e.g., default consistency, serial consistency, and fetch size) at both the session level and per-operation level using Spring property placeholders so that I can externalize environment-specific settings across development, test, and production.

As a developer, I want the configuration options to support both pooled and non-pooled connection settings, with configurable connection pooling parameters via Spring's environment abstraction, so that I can optimize resource usage based on application load and infrastructure constraints.
