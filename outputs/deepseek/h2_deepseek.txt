As a DevOps engineer, I want a single distributable archive (e.g., ZIP or TAR) containing all necessary server binaries, scripts, and default configurations so that I can deploy the Spring XD server consistently across environments.

As an operator, I want the distribution to include separate startup scripts for single-node and distributed modes (e.g., xd-singlenode and xd-admin/xd-container) so that I can choose the appropriate deployment topology at launch time.

As a system administrator, I want the packaged distribution to include all required runtime dependencies (JARs) and a preconfigured logging setup so that the server runs out-of-the-box without additional downloads or manual classpath configuration.

As a release engineer, I want the build process to produce versioned distribution artifacts with a clear directory structure (bin, lib, conf, logs) so that I can easily identify, version, and roll back releases.

As an end-user, I want the distribution to include sample configuration files and a quick-start guide (or README) so that I can verify the installation and run a basic stream within minutes of unpacking.
