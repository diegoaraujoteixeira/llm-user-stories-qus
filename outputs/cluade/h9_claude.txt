As a developer, I want to configure Cassandra connections using JavaConfig (e.g., @Configuration classes) so that I can set up the project in a type-safe, code-based manner consistent with modern Spring practices.
As a developer, I want to configure Cassandra connections using an XML namespace so that I can integrate with existing XML-based Spring application contexts.
As a developer, I want configuration support for cluster and session setup (e.g., contact points, ports, keyspace) so that I can easily connect to my Cassandra cluster with minimal boilerplate.
As a developer, I want configuration integration points for CassandraTemplate and repository beans so that I can wire up data access components using standard Spring dependency injection.
As a developer, I want sensible default configuration options with the ability to override them so that I can get started quickly while retaining flexibility for advanced setups.
