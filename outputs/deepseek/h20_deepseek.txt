As a web developer, I want to configure FTP/SFTP/FTPS connections (host, port, credentials, and key-based authentication) and save them as connection profiles so that I can deploy my project files to remote servers securely and reuse the settings across multiple projects.
As a web developer, I want to publish selected files, folders, or the entire project to a remote server via a one-click deployment action, with the option to synchronize only changed files, so that I can quickly push updates without manual file transfers.
As a web developer, I want to use the "Remote" view to browse, upload, download, rename, and delete files on the connected remote server directly within the IDE so that I can manage remote files without switching to a separate FTP client.
As a web developer, I want the publishing system to include a synchronization (sync/diff) feature that compares local and remote files and highlights differences so that I can review changes before deploying and avoid overwriting critical files.
