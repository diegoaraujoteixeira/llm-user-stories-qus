As a developer, I want a managed Cassandra Session/Cluster factory so that connections to the Cassandra cluster are created and managed consistently without manual boilerplate.
As a developer, I want the session factory to support connection pooling and lifecycle management (e.g., startup, shutdown) so that resources are used efficiently and cleaned up properly.
As a developer, I want the session factory to support keyspace-aware session creation so that I can target specific keyspaces without manually managing session state.
As an operator, I want the session factory to support cluster-level configuration (e.g., load balancing policies, retry policies, authentication) so that I can tune connection behavior for my deployment environment.
As a developer, I want the session factory to integrate with Spring's exception translation so that low-level Cassandra driver exceptions are converted into consistent Spring DataAccessException types.
